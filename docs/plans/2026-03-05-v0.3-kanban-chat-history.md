# Fluxnote v0.3 — Design Document

**Date:** 2026-03-05
**Author:** Technical Director (Claude)
**Status:** Approved by CEO

---

## Scope

Three focused improvements to the MVP:

1. Chat history sidebar — always visible, rename, delete
2. Kanban dashboard — 3 columns, drag-and-drop, DB-persisted column state
3. Assignment upload fix — timeout hardening + faster extraction model

---

## Feature 1: Chat History Sidebar

**Problem:** History only appears after first message; no rename capability.

**Solution:**
- "Recent Chats" section always rendered; empty state: *"Your conversations will appear here"*
- Rename: hover → pencil icon → inline text input → Enter/blur saves to localStorage
- Delete: unchanged (hover → trash icon)
- No backend change

**Files changed:** `Sidebar.jsx`, `Sidebar.css`, `api.js` (add `renameChatTitle`)

---

## Feature 2: Kanban Dashboard

**Problem:** Assignments displayed as a flat grid with no workflow progression.

**Solution:**
- 3 columns: To-Do | Currently Doing | Completed
- New assignments land in To-Do automatically
- Drag between columns → PATCH `/api/v1/assignments/{id}` saves `kanban_column` to DB
- Click card → existing AssignmentDetail modal (unchanged)
- Library: `@dnd-kit/core` + `@dnd-kit/sortable`

**DB migration (Supabase):**
```sql
ALTER TABLE assignments ADD COLUMN IF NOT EXISTS kanban_column text DEFAULT 'todo';
```

**Backend changes:**
- `app/models/assignment.py`: add `kanban_column: Optional[str]` to `AssignmentUpdate`
- No new endpoints required; existing PATCH handles it

**Frontend changes:**
- `DashboardView.jsx`: replace grid with KanbanBoard component
- `KanbanBoard.jsx` (new): DndContext wrapper with 3 KanbanColumn components
- `KanbanColumn.jsx` (new): SortableContext per column
- `KanbanBoard.css` (new): column layout styles

**What does NOT change:** AssignmentCard.jsx, AssignmentCard.css, AssignmentDetail.jsx, AssignmentDetail.css

---

## Feature 3: Assignment Upload Timeout Fix

**Problem:** AI extraction occasionally exceeds Vercel's serverless function timeout; DOCX files fail silently.

**Solution:**
- `assignment_extractor.py`: wrap httpx call in `asyncio.wait_for(..., timeout=45.0)` — raises `TimeoutError` cleanly
- `assignment_extractor.py`: switch from `MODEL_BALANCED` to `MODEL_FAST` (3–5× faster extraction)
- `assignments.py`: catch `asyncio.TimeoutError` explicitly, mark card as `failed` with message "Extraction timed out — please retry"
- `vercel.json`: already has `maxDuration: 60`; no change needed

---

## Architecture Constraints

- No new API endpoints
- No changes to any existing CSS files
- Backwards-compatible DB change (DEFAULT value means existing rows unaffected)
- Kanban column state is source-of-truth in DB (not localStorage)

---

## Risk Register

| Risk | Mitigation |
|---|---|
| @dnd-kit install breaks other deps | Pin exact versions, test locally |
| Supabase migration adds column | `IF NOT EXISTS` guard prevents errors on re-run |
| Fast model produces lower quality extraction | Fast model still produces structured JSON; quality acceptable for MVP |
| Rename overwrites chat title accidentally | Only saves on explicit Enter or blur; original title shown in input placeholder |
