# Fluxnote AI — Backend

FastAPI backend for the Fluxnote AI workspace. Provides AI chat (via OpenRouter), file upload + parsing, and workspace data management (notes, tasks, calendar events).

---

## Tech Stack

| Layer | Technology |
|---|---|
| Framework | Python 3.11 + FastAPI |
| Database | Supabase (PostgreSQL) |
| AI Models | OpenRouter (multi-model routing) |
| Streaming | Server-Sent Events (SSE) |
| Deployment | Railway (primary) / Vercel (alternative) |

---

## Setup (Local Development)

### 1. Prerequisites
- Python 3.11+
- A Supabase project (free at supabase.com)
- An OpenRouter account + API key (free at openrouter.ai)

### 2. Install dependencies
```bash
cd backend
python -m venv venv
source venv/bin/activate        # Mac/Linux
pip install -r requirements.txt
```

### 3. Configure environment
```bash
cp .env.example .env
# Edit .env with your real API keys
```

Required values in `.env`:
```
OPENROUTER_API_KEY=sk-or-...
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_KEY=your-anon-key
```

### 4. Set up the database
1. Go to your Supabase project → SQL Editor
2. Copy and paste the contents of `schema.sql`
3. Click Run

### 5. Run the server
```bash
uvicorn app.main:app --reload
```

Server will be available at: `http://localhost:8000`
Interactive API docs: `http://localhost:8000/docs`

## API Reference

### Base URL
- Local: `http://localhost:8000`
- Production: Your Railway deployment URL

All endpoints use `session_id` (a string UUID generated by the frontend) to scope data per user session.

---

### Chat

| Method | Endpoint | Description |
|---|---|---|
| POST | `/api/v1/chat/message` | Send a message, receive streaming AI response |
| GET | `/api/v1/chat/history` | Get chat history for a session |
| DELETE | `/api/v1/chat/history` | Clear chat history for a session |

**POST /api/v1/chat/message**
```json
{
  "message": "Summarise this document",
  "model": "Fast",
  "file_ids": ["uuid-of-uploaded-file"],
  "session_id": "browser-generated-uuid"
}
```
Returns: `text/event-stream` SSE with events:
- `{"type": "start"}`
- `{"type": "chunk", "content": "word by word text..."}`
- `{"type": "done"}`
- `{"type": "error", "message": "..."}` on failure

---

### Files

| Method | Endpoint | Description |
|---|---|---|
| POST | `/api/v1/files/upload?session_id=...` | Upload a file (multipart) |
| GET | `/api/v1/files?session_id=...` | List all uploaded files |
| DELETE | `/api/v1/files/{file_id}?session_id=...` | Remove a file |

**Supported formats:** PDF, DOCX, TXT, CSV, PPTX
**Limits:** 20MB per file, 5 files per session

---

### Workspace

| Method | Endpoint | Description |
|---|---|---|
| GET | `/api/v1/workspace/notes?session_id=...` | List notes |
| POST | `/api/v1/workspace/notes?session_id=...` | Create note |
| PUT | `/api/v1/workspace/notes/{id}?session_id=...` | Update note |
| DELETE | `/api/v1/workspace/notes/{id}?session_id=...` | Delete note |
| GET | `/api/v1/workspace/tasks?session_id=...` | List tasks |
| POST | `/api/v1/workspace/tasks?session_id=...` | Create task |
| PUT | `/api/v1/workspace/tasks/{id}?session_id=...` | Update task |
| DELETE | `/api/v1/workspace/tasks/{id}?session_id=...` | Delete task |
| GET | `/api/v1/workspace/events?session_id=...` | List calendar events |
| POST | `/api/v1/workspace/events?session_id=...` | Create event |
| PUT | `/api/v1/workspace/events/{id}?session_id=...` | Update event |
| DELETE | `/api/v1/workspace/events/{id}?session_id=...` | Delete event |

## AI Model Tiers

| UI Label | OpenRouter Model (default) | Characteristics |
|---|---|---|
| Fast | mistralai/mistral-7b-instruct:free | Quick responses, good for simple tasks |
| Balanced | google/gemma-2-9b-it:free | Better reasoning, moderate speed |
| Deep Think | deepseek/deepseek-r1-distill-llama-70b:free | Best reasoning, slower |

To swap models: update `MODEL_FAST`, `MODEL_BALANCED`, `MODEL_DEEP_THINK` in your `.env`.

## Deployment (Railway)

1. Push the `backend/` folder to GitHub
2. Create a new Railway project → Deploy from GitHub repo
3. Set root directory to `backend`
4. Add all environment variables from `.env` in Railway's Variables tab
5. Railway will auto-detect Python and use `railway.toml` for the start command

After deploy, copy the Railway URL and set it as `VITE_API_URL` in the frontend's `.env`.
